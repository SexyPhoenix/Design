#### 设计原则
###### 1. 开闭原则
对扩展开放，对修改关闭。
###### 2. 里氏代换原则
对实现抽象化的具体步骤的规范。
###### 3. 依赖倒转原则
针对接口编程，依赖于抽象而不依赖于具体。
###### 4. 接口隔离原则
使用多个隔离的接口，比使用单个接口要好
###### 5. 迪米特法则，又称最少知道原则
一个实体应当尽量少地与其他实体之间发生相互作用，使得系统功能模块相对独立。
###### 6. 合成复用原则
尽量使用合成/聚合的方式，而不是使用继承。
